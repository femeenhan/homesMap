# 도식화 다듬기 2 — 탑뷰 직접 수납장 편집 + 수납장 2-pane

작성일: 2026-07-24
근거: 사용자 피드백 — ① 탑뷰 편집에서 수납장이 직접 안 움직임(직접 조작 원함), ② 수납장 화면이 아래로만 펼쳐져 우측 공간 낭비 → 물건 목록을 우측 페인으로.

## 1. 확정 결정 (사용자 합의)

| 항목 | 결정 |
|---|---|
| 탑뷰 편집 | **수납장 직접 드래그 이동/리사이즈 허용**(방-로컬 셀 스냅, 방 안 클램프). 방 재탭 확대(RoomEditView)는 정밀 조작 옵션으로 유지 |
| 수납장 화면 | **2-pane, 모바일·데스크톱 공통**(칸 이름이 짧아 좁은 폭에서도 성립) — 좌: 칸 트리(아코디언, 물건 인라인 없음), 우: **선택 칸의 직속 물건 목록** |
| 선택 | 기본 = 수납장 루트(직속 물건). 칸 행 탭 = 선택 + 펼침 토글. 선택 칸 삭제·소실 시 루트로 복귀 |
| 추가 | 좌측 상단 `＋ 칸`(선택 컨테이너 하위에 추가), 우측 상단 `＋ 물건`(선택 컨테이너에 추가) — 기존 칸/물건 토글 폼은 이 화면에서 대체 |

## 2. 상세

- **탑뷰 수납장 편집**: 방 내부 셀은 정사각이 아님(가로=방폭/12, 세로=방높이/rows) → `EditableTile`에 `cellH?: number`(기본=cell) 추가, 이동 dy·top/height 계산에 사용. 편집 모드의 수납장 오버레이(현재 `pointer-events:none` div)를 EditableTile로 승격 — 중첩 타일이므로 `onClick`에 `stopPropagation` 추가(수납장 탭이 방 선택으로 번지지 않게), pointerdown은 기존 stopPropagation으로 방 드래그와 이미 분리됨. `.gm-edit .gm-sto{pointer-events:none}` 규칙 삭제. 선택 상태는 방/수납장 공용 `selectedId` 하나. 힌트 문구: "수납장은 바로 드래그, 방을 한 번 더 탭하면 확대 편집".
- **2-pane**: `StorageScreen` 재작성 — `DrillHeader` 아래 좌/우 분할(좌 ~40%·우 나머지, 각자 세로 스크롤).
  - 좌: 루트 행(수납장 이름+직속 개수, 탭=루트 선택) + 칸 트리(재귀, `TreeRow` 재사용 — 탭=선택+펼침, ⋯ 이름수정/삭제, 직속 개수 뱃지, 선택 행 하이라이트 `sel` 클래스). 상단 `＋ 칸`(`InlineInput`) → 선택 컨테이너 하위.
  - 우: 상단 `＋ 물건`(`InlineItemForm` — 이름+메모·사진) → 선택 컨테이너. 아래 `ItemRow` 목록(photoUrls 썸네일). 빈 상태 "아직 물건이 없어요".
  - `CompartmentTree`는 이 화면에서 더 이상 사용 안 함(목록 뷰 전용으로 잔존).
- 검색 점프 flash(`gm-focus`)는 우측 페인 타깃으로 조정.

## 3. 범위 외

칸 간 물건 이동(드래그), 하위 포함 전체 물건 보기 토글, 수납장 방 간 이동.

## 4. 검증

`tsc`/`eslint`/`vitest`/`build` 통과. 수동: 탑뷰 편집에서 수납장 드래그/리사이즈(+방 확대 병행), 2-pane 선택/추가/삭제 흐름(모바일 390px 포함), 검색 점프.
