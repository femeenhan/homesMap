# 목록 트리 "심플 정돈" 설계 — 참고 이미지(홈즈 MVP 심플) 반영

작성일: 2026-07-23
근거: 사용자가 `docs/simple_ui.png`("홈즈 MVP – 심플 버전")를 참고로 제시. "똑같이는 아니고, 이런 형태의 심플·깔끔·직관 디자인." Phase 1(터치 우선 아코디언 트리) 배포 위에 얹는 **시각/입력 정돈** 라운드.

범위: 목록(트리) 뷰. 데이터 모델·저장·도식화·테마 토큰 무변경.

## 2. 목표 / 비목표

**목표:** 현재 다크 아코디언 트리를 참고 이미지처럼 **심플·깔끔·직관**하게 — 시각 노이즈를 줄이고, 추가를 한 곳(상단)으로 모으고, 입력을 이름 하나로 최소화.

**비목표:** 내비게이션 구조 변경, 테마 색 변경, 지도 재설계, 물건 상세 화면, 모달 입력, 데이터/저장/콜백 변경. 이전 계획의 "데스크톱 2-pane·사진 그리드"는 이 정돈 라운드에서 **보류**(사용자가 아코디언·단순 방향 선택).

---

## 3. 시각 정돈 (무조건 포함 — 구조·테마 안 건드림)

Phase 1의 기능은 유지하되 룩앤필만 참고 이미지의 차분함으로.

- **행 노이즈 감소:** 현재 행 `[caret][아이콘][이름][개수][＋][⋯]` → `[caret][아이콘][이름] … [개수][⋯]`. **행에서 `＋` 제거**(상단으로 이동, §4). 개수 뱃지는 은은하게(작게·저채도), 없으면 숨김.
- **여백/리듬:** 행 간 호흡 확보(터치 44px 유지, 데스크톱 조밀 유지). 아이콘·이름·⋯ 정렬 정돈.
- **아이콘 톤:** 방 🏠 / 수납장 📦 / 칸 📁 / 물건은 상자 아이콘. 참고 이미지처럼 폴더=컨테이너, 상자=리프로 시각 구분 일관.
- **라운드/타이포:** Clay 토큰 정합 — 카드/입력 라운드 12–16px, 이름 14px, 메타 11–12px, 코랄 액센트. 새 색 도입 없음(기존 `--` 변수만).
- **빈 상태:** "아직 ○○이 없어요 — 위 `＋ 추가`로 시작" 톤 통일.
- **트리 컨테이너:** `.tree-view`/`.home-tree` 여백·최대폭 정돈(현행 620px 유지), 배경 카드감 은은하게.

---

## 4. 상단 `＋ 추가` (인라인 `＋` 대체)

참고 이미지: 각 패널 맨 위에 `＋ 추가`(root는 `＋ 새 공간`). 아코디언에 매핑:

- **행에서 우측 인라인 `＋`(및 `＋칸`/`＋물건`) 제거.** 행 우측엔 `⋯`(이름수정/삭제)만 남김 → 행이 훨씬 깔끔.
- **노드를 펼치면, 자식 목록 맨 위 첫 행 = `＋ 추가`** (들여쓰기는 자식 레벨). Root도 `＋ 방 추가`를 목록 **맨 위**로 이동(현재는 맨 아래).
- **레벨별 추가 대상:**
  - Root → `＋ 방 추가`
  - 방 → `＋ 수납장 추가`
  - 수납장·칸 → `＋ 추가` (하위가 칸/물건 2종이므로 §5의 종류 토글로 구분)
- **동작:** `＋ 추가` 탭 → 그 자리 인라인 입력(§5)이 펼쳐짐. 기존 `InlineInput`/`InlineItemForm` 재사용.

효과: 추가 진입점이 "각 레벨 맨 위 한 곳"으로 일관·예측 가능. 행마다 붙던 `＋`가 사라져 목록이 조용해짐.

---

## 5. 최소 입력 (이름만) + 메모/사진 보존

참고 이미지 원칙 "입력은 최소(이름만)". 단 기존 메모/사진 기능은 없애지 않음.

- **방/수납장/칸 추가:** `이름` 한 칸 + `추가`/`취소`. (이름만 — 현행과 사실상 동일, 더 정돈.)
- **수납장·칸의 `＋ 추가`:** `이름` + **종류 토글 `칸 / 물건`** (참고 이미지의 "종류 선택"에 대응, 드롭다운 대신 2-토글). 선택에 따라:
  - `칸` → 이름만.
  - `물건` → 이름 + **`＋ 메모·사진`(선택, 접힘)**. 기본은 이름 한 칸만 보이고, 누르면 메모/사진 필드 펼쳐짐. → **기본 최소 입력, 메모/사진은 보존.**
- **재사용:** `InlineItemForm`을 "이름 필드 + 접힘 메모·사진" 형태로 정돈(필드 자체는 기존 것).

이름 수정·삭제는 **Phase 1의 `⋯` 메뉴 그대로**(모달 도입 안 함). 삭제 확인 모달 카피만 참고 이미지 톤으로 정돈("'○○'을 삭제하시겠습니까? 하위 항목도 함께 삭제됩니다").

---

## 6. 위치 브레드크럼

- **검색 결과**에 위치 경로 표시: `양말 — 안방 › 붙박이장 › 서랍 › 첫째칸`. (참고 이미지 §4.)
- 경로 계산: 물건→칸(상위 칸 체인)→수납장→방. `lib/`에 경로 헬퍼가 있으면 재사용, 없으면 단순 조립 유틸 1개.
- 물건 상세 화면은 이번 범위 밖이므로 브레드크럼은 **검색 결과에만** 우선 적용.

- **변경 예상:** `HomeTree.tsx`(상단 ＋ 추가 배치, 행에서 ＋ 제거), `TreeRow.tsx`(행 우측 ＋ 제거 → ⋯만; `addOptions` prop 제거 또는 상단 추가로 이전), `CompartmentTree.tsx`(CompartmentNode 상단 ＋, InlineItemForm 접힘 메모·사진), `globals.css`(트리 시각 정돈), 검색 컴포넌트(`SearchBar`/결과)에 브레드크럼.

## 8. Phase 1 대비 바뀌는 것 (명시)

- Phase 1의 **행 우측 인라인 `＋`(＋칸/＋물건) → 제거**, 각 레벨 **상단 `＋ 추가`**로 이전. (사용자가 이전엔 인라인 ＋를 택했으나, 참고 이미지 기준 상단 ＋로 갱신.)
- 물건 추가 폼: 메모/사진 **상시 노출 → 접힘 선택**(최소 입력).
- 나머지 Phase 1 인터랙션(탭=펼치기, ⋯ 이름수정/삭제, 44px, hover 분기)은 **유지**.

---

## 9. 열린 항목 / 확인 필요

- **메모/사진 처리:** "이름만 + `＋ 메모·사진` 접힘"으로 보존하는 안(§5). 만약 메모/사진을 이번에 아예 빼길 원하시면 축소 가능(단 기존 데이터 표시는 유지).
- **종류 토글 vs 두 버튼:** 수납장/칸의 `＋ 추가`를 단일 진입+`칸/물건` 토글로 제안(§5). "＋칸/＋물건 두 버튼 상단 배치"가 더 직관적이면 그쪽도 가능.
- **개수 뱃지 유지 여부:** 참고 이미지엔 개수 표시가 없음. 유용성 위해 은은하게 유지 제안하나, 완전 제거해 더 깔끔하게도 가능.
