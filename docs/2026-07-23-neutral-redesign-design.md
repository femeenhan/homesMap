## 2. 디자인 토큰 (전역 교체)

`globals.css :root` 변수 **값만** 교체 — CSS 변수라 맵·헤더 포함 전 화면에 자동 반영. 맵 코드는 보류지만 색이 같이 바뀌는 것은 의도된 결과.

| 토큰 | 라이트 | 다크 |
|---|---|---|
| `--bg` (페이지) | `#ffffff` | `#141517` |
| `--panel` (인셋/카드, **신규**) | `#f7f7f8` | `#1e1f22` |
| `--surface` (입력/모달) | `#ffffff` | `#232428` |
| `--line` (보더) | `#e4e4e7` | `#2c2e33` |
| `--ink` (본문) | `#1f2328` | `#e8eaed` |
| `--ink-soft` (보조) | `#6f7680` | `#9aa0a6` |
| `--accent` | `#2563eb` | `#5b8def` |
| `--accent-soft` (hover/선택 배경) | `#eff4ff` | `#1e2836` |
| `--danger` | `#dc2626` | `#ef6a6a` |

- **보더**: 현행 2px → **1px** 헤어라인. 다크모드는 `@media (prefers-color-scheme: dark)` 현행 방식 유지.
- **라운드**: 카드/입력 12–16px → **8–10px**, 버튼 8px, 칩류 6px.
- **그림자**: 최소화 — 플랫 + 헤어라인 보더가 기본. 모달/드롭다운만 은은한 그림자.
- **타이포**: 체계 유지 — 행 이름 14px, 메타/브레드크럼 12px, 화면 타이틀 16–17px semibold. 시스템 폰트 유지.
- 기존 토큰명은 최대한 유지(값 교체). 안 쓰게 되는 토큰(coral 계열)은 삭제.
- `--panel` 신규 사유: 현행은 `--bg`가 페이지 바닥과 인셋(칩·썸네일) 배경을 겸하는데, 페이지가 흰색이 되면 인셋용 회색이 분리 필요. 인셋 용법의 `--bg` 사용처만 `--panel`로 치환.

## 3. 아이콘 시스템

새 의존성 없이 **`components/Icon.tsx` 하나**에 인라인 SVG(lucide 패스, `stroke-width:1.5`, `stroke:currentColor`, `fill:none`, 기본 18px):

| 아이콘 | 용도 |
|---|---|
| `folder` | 방·수납장·칸 (모든 컨테이너 공통 — 이미지 규칙) |
| `box` | 물건 (리프) |
| `search` | 검색 입력 |
| `plus` | 상단 ＋ 추가 |
| `chevron-left` | 모바일 뒤로가기 |
| `chevron-right` | 아코디언 캐럿(펼침 시 90° 회전)·드릴다운 행 우측 |
| `more-horizontal` | ⋯ 행 메뉴 |
| `x` | 닫기·취소 |
| `trash` | 물건 행 삭제 버튼 |
| `camera` | 사진 첨부·사진 있음 표시 |
| `check` | 사진 선택됨 |
| `clock` | 헤더 가족 활동 버튼 |

교체 대상: 헤더 로고 `🔍🏠` → **텍스트만**(마크 제거, 이미지 §1 스타일), 활동 버튼 🕘 → clock, 초대 버튼 → 텍스트만, 트리 행 🏠📦📁, 종류 토글 `📁 칸/📦 물건` → 텍스트만, 검색 썸네일, 물건 아이콘(`item.emoji || '📦'` → 박스 아이콘 고정 — 생성 시 하드코딩뿐이라 선택 UI 없음, **데이터 필드는 무변경**).
유지: 멤버 아바타 emoji(사용자 데이터). 맵 내부 이모지는 맵 라운드에서.

## 4. 모바일(<768px) — 드릴다운

이미지 §1 패턴. 한 화면 = 한 레벨:

- **루트**: 헤더(검색 포함) → `＋ 방 추가` → 방 목록.
- **행**: `[folder] 이름 ……… [chevron-right] [⋯]` — 행 탭=진입, ⋯=이름수정/삭제.
- **하위 화면**: 상단 `[‹] 현재 이름 … [⋯]` → `＋ 추가` → 하위 목록. 수납장→칸→물건 동일 패턴 반복.
- **물건 행**: `[box] 이름 … [⋯]` — 탭 시 기존 DetailPanel(재설계는 다음 라운드).
- **상태**: 컴포넌트 state로 선택 경로 스택(예: `[roomId, storageId, cmpId…]`). `matchMedia('(max-width: 767px)')` 분기.
- ponytail: 안드로이드 백버튼/브라우저 히스토리 연동 스킵 — 화면 내 `‹`만. 불편 확인되면 다음에 history 연동.

## 5. 데스크톱(≥768px) — 아코디언 유지

구조·인터랙션 무변경: 탭=펼치기, 각 레벨 상단 `＋ 추가`, ⋯ 메뉴, 44px 터치 타깃. 토큰·아이콘 교체만으로 뉴트럴 룩 적용.

## 6. 폼 · 삭제 · 검색 (재스킨)

- **추가/수정**: 기존 인라인 폼 유지(모달 아님). 이름 한 칸 + 추가/취소. 수납장·칸의 종류 토글 `칸/물건` 유지 — 이모지 제거, 텍스트+아이콘. 물건의 `＋ 메모·사진` 접힘 유지.
- **삭제 확인**: 기존 Modal 재스킨. 카피 유지("'○○'을 삭제하시겠습니까? 하위 항목도 함께 삭제됩니다"). 삭제 버튼 `--danger`.
- **검색**: 결과 브레드크럼(기구현) 재스킨 — `이름` 14px + 경로 12px `--ink-soft`, 구분자 `›`. 썸네일 이모지 → 박스 아이콘.
