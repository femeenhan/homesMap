# 홈즈맵 목록(트리) 뷰 재설계 — 노션식 반응형 트리

작성일: 2026-07-23
범위: 목록(도식화 아님) 뷰의 인터랙션·레이아웃 전면 교체. 데이터 모델·저장·도식화 뷰 무변경.

---

## 1. 배경 — 무엇이 왜 깨졌나

현재 트리는 **데스크톱 파일탐색기 패턴**으로 구현돼, 터치에서 핵심 동작 3개가 전부 어긋난다.

- **이름이 상시 편집 `<input>`** (`HomeTree.tsx:54,88`, `CompartmentTree.tsx:160`, `globals.css:311`).
  행의 70%를 차지하는 이름을 탭 → 즉시 rename 모드 + 소프트키보드 팝업. blur 후 뷰포트 원복 안 됨.
  → 사용자 불만 "탭하면 이름 수정되고 화면 포커스 됐다가 안 돌아옴"의 직접 원인.
- **"안 보기"(펼치기)가 14px caret/아이콘에만 매핑** (`globals.css:309`). 주 동작이 가장 작은 타겟에.
- **모든 추가/삭제 버튼이 `:hover`에서만 노출** (`globals.css:318-319,336-337`). 터치엔 hover 없음 →
  `＋수납장·＋칸·＋물건·🗑️`가 모바일에서 사실상 안 보임. → "추가가 너무 불편"의 직접 원인.
- 행 높이 30px, 버튼 폰트 11px — 44px 터치 최소치 미달.

결론: 부분 수정이 아니라 **행 인터랙션 모델을 터치 우선으로 재설계**한다.

## 3. 목표 / 비목표

**목표**
- 탭 = "안 보기"(펼치기/상세). 이름 수정은 명시적 동작으로만.
- 모든 컨트롤 ≥ 44px, hover 의존 0.
- 추가/수정/삭제가 모바일에서 한눈에 보이고 쉬움.
- 데스크톱은 노션식 2-pane으로 공간 활용.

**비목표**
- 도식화(지도) 뷰 — 범위 밖, 그대로 병존.
- 데이터 모델·`store`/`sync`·콜백 시그니처 변경 — UI 트리거만 교체.
- 검색/가족초대/활동피드 등 다른 화면.

## 4. 반응형 레이아웃

브레이크포인트: **`min-width:900px`** (기존 640px는 시트용이라 재사용 안 함).

- **넓은 화면(≥900px)**: 좌측 트리 + 우측 상세/입력 pane 2단.
  - `.tree-view`를 `display:flex`로, 좌(트리, `max-width:420px`) + 우(pane, `flex:1`).
- **좁은 화면(<900px)**: 트리 단일 pane. 상세·입력은 인라인 확장. 우측 pane 미렌더.

선택 상태(`selectedNodeId`, `selectedNodeType`)는 `HomeTree`가 보유. 넓은 화면에서만 우측 pane에 반영.

## 5. 트리 행 인터랙션 명세

행 앵커: `[▸/▾ caret][아이콘][이름=span][개수]  ……flex-gap……  [＋][⋯]`

### 5.1 컨테이너 행 (방 / 수납장 / 칸)
- **행 몸통 탭 = 펼치기/접기.** 데스크톱에선 동시에 **선택**(우측 pane 갱신).
- `▸/▾`는 상태 **표시**일 뿐(회전 애니메이션). 탭 타겟은 행 전체(≥44px).
- **이름 = `<span>` 텍스트.** 탭해도 편집 안 됨/키보드 안 뜸.
- **우측 인라인 컨트롤**:
  - `＋` = 하위 추가 트리거. 방→`＋수납장` 직행 인라인 입력. 수납장/칸→`＋칸`·`＋물건` 두 버튼 노출.
  - `⋯` = 메뉴(이름 수정 / 삭제).
  - 노출 규칙: `@media (hover:hover)` → hover 시 노출(노션 데스크톱 동일). 아니면(터치) **항상 노출.**
  - `＋`/`⋯`는 `stopPropagation`으로 행 탭과 분리.

### 5.2 물건 행 (리프)
- **데스크톱**: 클릭 → 우측 pane에 상세/편집.
- **모바일**: 탭 → 행 아래 **인라인 상세 확장**(사진·메모·등록자·수정·삭제).
- **사진 탭 → 전체화면 라이트박스**(데스크톱/모바일 공통).
- 우측 `⋯` = 수정 / 삭제.

## 6. 추가 흐름

- **방**: 루트 `＋ 방 추가`(현행 유지) + 각 방 행 `＋` → 인라인 `수납장 이름` 입력.
- **수납장 / 칸**: 행 `＋` → `＋칸`·`＋물건` 두 버튼. 
  - `＋칸` → 인라인 이름 입력(`InlineInput`).
  - `＋물건` → 이름·메모·사진 폼(`InlineItemForm`):
    - **데스크톱** → 우측 pane에 폼.
    - **모바일** → 해당 행 아래 인라인 폼. 열릴 때 `scrollIntoView({block:'nearest'})`로 부드럽게 보이게(포커스 튐 방지).
- 추가 버튼은 터치 사이즈업(패딩↑, ≥44px 히트박스).

---

## 7. 이름 수정 / 삭제

- `⋯` → **이름 수정** / **삭제**.
- **이름 수정** → 그 행의 이름만 `<input>`으로 전환(autofocus). Enter/blur 확정, Esc 원복. **명시적일 때만 편집.**
- **삭제** → 확인 단계(기존 `DeleteBtn` 2단계 또는 `Modal` 재사용). "하위 칸·물건 포함 삭제" 경고 유지.
- 메뉴 표면:
  - 모바일 → 바텀 액션시트(기존 `.sheet-wrap`/`.sheet` 재사용).
  - 데스크톱 → 앵커 팝오버(작은 드롭다운).

---

## 8. 데스크톱 우측 pane 명세

- **컨테이너(방/수납장/칸) 선택** → 헤더(이름 + 경로 브레드크럼) + **물건 사진 그리드**.
  - 카드 = 사진 썸네일(없으면 이모지) + 이름 + 메모.
  - 그리드 끝에 `＋ 물건 추가` 카드.
  - 카드 클릭 → 물건 선택(상세로). 사진 클릭 → 라이트박스.
- **물건 선택** → 큰 사진 + 편집 필드(이름/메모/등록자 표시) + 삭제.
- **미선택** → 빈 상태 안내("왼쪽에서 항목을 선택하세요").

---

## 9. 사진 뷰어 (라이트박스, 데스크톱·모바일 공통)

- 트리거: 썸네일/인라인 상세/그리드 카드의 사진 탭·클릭.
- 전체화면 오버레이(사진 크게, `✕` 닫기, 배경 탭 닫힘). `position:fixed;inset:0`.
- 복호화/objectURL 로딩은 `DetailPanel`의 기존 로직 재사용(§11).

## 11. 사진 복호화 로직 재사용

`DetailPanel.tsx:39-79`의 지연 로드/objectURL 캐시/언마운트 해제 로직을 `PhotoGrid`·인라인 상세·`Lightbox`가 공유해야 함.
- 1차: 공통 훅 `usePhotoUrls(items)`로 추출(반환: `photoUrls`, `photoErrors`, 삭제 시 정리 함수).
- `DetailPanel`도 이 훅으로 치환(중복 제거) — 단, 도식화 뷰 회귀 없도록 동작 동일 검증.
- `ponytail`: 훅 추출이 과하면 트리 쪽에 일단 이식, 3곳 이상 중복 확인되면 훅화.

## 12. CSS 명세 (globals.css 트리 섹션 재작성)

- 행: 높이 44px, `＋`/`⋯` 히트박스 ≥44px, `@media (hover:hover)`에서만 hover-노출·아니면 상시.
- caret: 표시용 회전 인디케이터(`transform:rotate`), 탭 타겟은 행.
- 2-pane: `@media (min-width:900px)`에서 `.tree-view` flex 2단.
- 인라인 확장 폼/상세, 사진 그리드(`grid-template-columns:repeat(auto-fill,minmax(...))`), 라이트박스(`position:fixed`).
- 들여쓰기 모바일 16→14px, 깊이 깊어도 이름 폭 확보.

## 13. 단계적 롤아웃 (권장)

핵심 통증(모바일)부터 독립 배포 가능하도록 나눔:

- **Phase 1 — 터치 우선 트리(모바일 통증 해결)**: 탭=펼치기, 이름 span화, ＋/⋯ 상시노출, 44px, 인라인 추가/수정/삭제, `TreeRow` 통합. *여기까지만으로 사용자 불만 3개 전부 해소.*
- **Phase 2 — 데스크톱 2-pane + 사진 그리드**: `≥900px` pane, `DetailPane`, `PhotoGrid`.
- **Phase 3 — 라이트박스 + 사진 로직 훅화**: `Lightbox`, `usePhotoUrls` 공통화.

각 Phase는 tsc/eslint 통과 후 커밋·배포(홈즈맵 배포 정책).

## 14. 검증

- `npm run lint`, `tsc`(빌드) 통과.
- 수동 시나리오(모바일·데스크톱 각):
  1. 방→수납장→칸→물건 추가가 hover 없이 전부 가능.
  2. 행 탭 시 rename/키보드 안 뜸, 펼침만.
  3. `⋯ → 이름 수정`만 편집 진입, Esc 원복.
  4. 삭제 확인 후 하위 함께 삭제.
  5. 사진: 데스크톱 그리드 표시 / 모바일 탭 시 전체화면.
- 도식화 뷰 회귀 없음(공유 로직 훅화 시 특히).

## 15. 열린 항목 / 리스크

- `TreeRow` 통합 시 방/수납장/칸의 미세한 차이(아이콘, 자식 타입, 추가 버튼 종류)를 prop으로 분기 — 과분기되면 가독성↓. 통합 후 재검토.
- 우측 pane과 트리가 같은 물건을 이중 표시(중복감) — 노션식 마스터-디테일의 정상 특성으로 수용.
- 사진 다수 시 그리드/라이트박스 복호화 부하 — 지연 로드 유지, 필요 시 뷰포트 기반 로드.
