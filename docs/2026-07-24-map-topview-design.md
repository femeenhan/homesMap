# 도식화 개편 2 — 통합 탑뷰 + 수납장 아코디언

작성일: 2026-07-24
근거: 사용자 제안 — "방 레이아웃 위에 수납장 표기+편집, 수납장 탭 후엔 아코디언이 직관적". 레벨 3→2 축소.

## 1. 확정 결정 (사용자 합의)

| 항목 | 결정 |
|---|---|
| 구조 | **2레벨**: ① 우리집 탑뷰(방+수납장 통합 표시) ② 수납장 화면(아코디언). 기존 L1(방 확대 내비)·L2(칸 스택) 폐기 |
| 탑뷰 표시 | 방 사각형 안에 수납장 타일을 **비례 오버레이**(방-로컬 셀 좌표를 방 사각형에 스케일) — 참고 이미지 "우리집 탑뷰" |
| 보기 흐름 | 수납장 탭 → 수납장 화면. 방 영역 탭은 무동작(보기에서 방은 레이아웃일 뿐) |
| 수납장 화면 | `‹ 뒤로 + 이름 + ⋯` 헤더 + 상단 `＋ 추가`(칸/물건) + **`CompartmentTree` 아코디언 재사용**(photoUrls 포함) |
| 편집(확정: 옵션1) | 탑뷰 편집 = 방 이동/리사이즈. **선택된 방을 다시 탭 → 방 확대 편집 화면**(수납장 이동/리사이즈 + 방 이름수정/삭제 + ＋수납장 추가), `‹`로 탑뷰 편집 복귀 — 모바일 타깃 크기 문제 해결 |
| 검색 점프 | 해당 수납장 화면으로 직행 + 하이라이트(아코디언 영역 flash) |

## 2. 상세

- **탑뷰 수납장 오버레이**: `storageRect(s)`(방 내부 12×N 셀)를 방 px 사각형에 비례 변환해 절대배치. 이름 텍스트(작게, 말줄임), 보기 모드에서만 탭 가능. 편집(탑뷰) 중엔 `pointer-events:none`(방 드래그 방해 금지). 방 타일 `overflow:hidden`.
- **방 확대 편집(RoomEditView)**: 방 내부 그리드(`roomInnerGrid`)를 화면 폭으로. `EditableTile` 재사용(수납장 min 1×1). **행 확장 없음** — 수납장은 방 안에 있어야 하므로 `maxRows=inner.rows`로 이동·리사이즈 클램프(EditableTile에 `maxRows?` prop 추가). 헤더는 `DrillHeader` 재사용(이름수정/삭제 — 삭제 시 방 소실로 자동 탑뷰 복귀).
- **수납장 화면**: 수납장이 동기화로 사라지면 자동 탑뷰 복귀(id 조회 실패 → null). 칸/물건 추가·수정·삭제는 목록 뷰와 완전히 같은 컴포넌트·핸들러.
- **상태 단순화**: 맵 경로 스택(`resolvePath`) 불필요 → `storageId | null` + `editRoomId | null` 두 상태. `drillPath`는 목록 드릴다운 전용으로 잔존.
- **삭제**: `RoomCanvas`(L1)·`StorageView`(칸 스택)·`.gm-stack`/`.gm-block` CSS·맵의 resolvePath 사용.

## 3. 범위 외

수납장을 다른 방으로 드래그 이동(방 간 이동은 목록에서 불가한 것과 동일 — 다음 후보), 핀치 줌, 방 색상.

`tsc`/`eslint`/`vitest`/`build` 통과. 수동: 탑뷰에 수납장 보임, 수납장 탭→아코디언, 편집(방 이동/리사이즈, 선택-재탭 확대, 수납장 조작·클램프), 검색 점프, 라이트/다크·모바일/데스크톱.
