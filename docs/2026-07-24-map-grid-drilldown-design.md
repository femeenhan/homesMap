# 도식화 재설계 — 그리드 드릴다운 맵

작성일: 2026-07-24
근거: 사용자 요청 "도식화 구조적 개선 — 목록처럼 간소화, 이모지 제거, 모바일에서 방 크기조절 불가·상하 공백 문제 해결". 참고 이미지 `docs/simple_ui.png` §2(단순 오버레이 맵). 뉴트럴 리디자인(목록 라운드) 위에 얹는 라운드.

## 2. 그리드 월드

- **L0(우리집)**: 가로 **12칸 고정**, 세로 행은 동적 — 최소 행 = max(콘텐츠가 차지하는 행, 뷰포트 높이를 채우는 행 수), 방을 아래 경계로 옮기거나 늘리면 행 자동 확장. 위/좌/우 경계만 제한. 셀 한 변 = 컨테이너 폭 ÷ 12(정사각). 데스크톱은 max-width 720px 중앙.
- **방 좌표**: 기존 `x,y,w,h` 필드를 **셀 단위로 재해석**(스키마 무변경). 방 최소 2×2셀, 새 방 기본 4×3셀.
- **레거시 변환(1회)**: 저장값에 `w>12 || h>12`가 있으면 px(940×600 시절)로 판정 → 비례 변환(940px=12칸 기준, y도 동일 셀 크기로 환산) 후 저장. 변환 후엔 모든 값이 ≤12라 재변환 없음.
- **L1(방 확대) 그리드**: 방 내부를 가로 12칸 × 세로 `round(12·h/w)`행으로 확대(방 비율 유지). 수납장 좌표는 이 L1 셀 공간에 저장.
- **수납장 레거시 변환**: 방이 px 판정이면 그 방 수납장(전역 px 중심좌표)도 px로 간주 → 방 내부 비율 → L1 셀로 함께 변환·저장.

## 3. 드릴다운 3레벨

- **L0 우리집**: 방 사각형(이름 + 수납장 개수 뱃지). 탭=방 확대. 상단 `＋ 방 추가`.
- **L1 방 확대**: 상단 `‹ 뒤로 + 방 이름 + ⋯`(이름수정/삭제 — 목록 DrillHeader·RowMenu 재사용), 수납장 사각형 타일(이름 텍스트). 탭=수납장 확대. 상단 `＋ 수납장 추가`.
- **L2 수납장 확대**: 상단 동일 헤더. 칸 = **세로 자동 스택**(루트 칸 균등 분할, 좌표 없음·편집 없음), 칸 탭=그 칸 확대(중첩 칸 재귀, 같은 화면 패턴). 물건은 스택 아래 **목록**(기존 ItemRow) + `＋ 추가`(InlineAddForm — 칸 추가/물건 추가 토글 재사용).
- 경로 상태는 목록 드릴다운의 `resolvePath`/`PathSeg` 재사용(동기화로 노드 소실 시 조상 복귀 동일).
- **검색 연동**: 검색 결과 탭 → 도식화 탭 전환 + 해당 수납장 L2로 이동 + 하이라이트(블루 아웃라인, 기존 pulse 대체).

## 4. 보기/편집 모드

- **보기(기본)**: 탭=드릴다운만. 드래그 없음 → 스크롤 충돌 원천 제거.
- **편집**: L0·L1 상단 `편집` 버튼으로 진입, `완료`로 복귀.
  - 이동: 타일 드래그(셀 스냅, `touch-action:none`은 편집 모드에서만).
  - 리사이즈: 탭으로 선택 → 우하단 **44×44px 코너 핸들** 드래그(셀 스냅, 최소 크기 방 2×2·수납장 1×1).
  - 겹침: 허용(단순화 — 검증·충돌 밀어내기 없음). 아래 경계로 드래그 시 행 확장.
- **추가**: 각 레벨 상단 `＋ 추가` → 이름 인라인 입력(목록과 동일 폼) → **빈 영역 자동 배치**(좌상단부터 스캔, 없으면 새 행). 드래그 생성·팔레트 탭 배치 도구 폐기.
- 새 수납장 type은 `'box'` 고정(종류 팔레트 제거 — type 필드는 데이터로 잔존).

## 5. 수납장 타일 크기

- 기본 3×2셀(L1 셀 기준). 편집 모드 리사이즈 지원 → **storages에 `w`,`h` 컬럼 추가**(int, NULL 허용, NULL=기본 크기) — Supabase 마이그레이션 1개. 동기화(LWW)는 기존 행 단위 규칙 그대로.

## 6. 제거/정리 (코드 다이어트)

- 컴포넌트 폐기: `MapCanvas.tsx`, `DetailPanel.tsx`, `RoomDetail.tsx`, `StorageBadge.tsx`(신규 맵이 대체).
- `geometry.ts`: LOGICAL_W/H·fitScale·MAX_SCALE 등 px 월드 상수 제거, 신규 그리드 유틸로 대체(레거시 변환에만 940×600 상수 잔존).
- page.tsx: `Tool` 상태·좌측 툴바·storage 팔레트·드래그 생성 핸들러 제거.
- CSS: `.toolbar`·`.mode-btn`·`.pal-btn`·`.storage`(뱃지)·`.detail-panel`·`.rd-*`·pulse 등 폐기 셀렉터 삭제, 맵 푸터 이모지 제거(카운트는 텍스트만 유지), `.map.empty::after` ✏️ 카피 교체.
- ROOM_COLORS·STORAGE_TYPES(em·presets) UI 사용처 제거(타입 정의는 데이터 호환 위해 잔존 가능).

## 7. 검증

- 그리드 변환·자동 배치·레거시 판정은 순수함수로 분리 + vitest(TDD).
- `tsc`/`eslint`/`vitest` 통과, `npm run build` 통과.
- 수동: 모바일 390px(보기 드릴·편집 이동/리사이즈·상하 공백 없음), 데스크톱 1024px, 라이트/다크, 검색→L2 점프, 레거시 데이터 1회 변환.

## 8. 호환성 주의

- 좌표 재해석은 **클라이언트 전원 새 버전 전제**(PWA 캐시 특성상 구버전이 잠시 남을 수 있음 — 구버전이 셀 좌표를 px로 그리면 좌상단에 몰려 보이나 데이터 파손은 없음, 과도기 수용).
